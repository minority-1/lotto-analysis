# 고급 분석 코드 리뷰

검토일: 2026-07-17

## 범위

* 번호쌍·3개 조합·동반 출현과 회차 간 관계
* 7×7 행렬, 대각선과 기간 차이 행렬
* AC·간격·소수·합성수·제곱수·합계 구간·끝수 합계
* 과거 당첨 조합 중복과 Jaccard 유사도
* PostgreSQL Repository를 사용하는 분석 Service와 CLI
* 관련 결과 모델, JSON 내보내기와 테스트

## 결론

P0 또는 P1 발견 사항은 없다. 전체 1232회차에서 핵심 합계 불변식과 PostgreSQL 실행 결과가 일치했으며, 현재 규모에서 기능을 사용할 수 있다.

번호 생성 단계로 넘어가기 전에 P2의 범위 검증을 보완하는 것을 권장한다. P3 항목은 번호 생성 CLI를 추가하기 전 구조 정리와 백테스트 설계 시 함께 처리할 수 있다.

## 발견 사항

### P2: 보유 회차를 초과한 `--recent` 요청을 구분하지 않는다

상태: 2026-07-17 해결. 공통 Service 범위 로더가 요청 건수와 실제 반환 건수를 비교하고 명시적 오류를 반환한다.

Repository의 `list_draws(recent=N)`은 N이 보유 건수보다 커도 가능한 전체 행을 반환한다. 단일 기간 분석 Service는 실제 반환 건수와 요청 건수를 비교하지 않으므로 `patterns --recent 5000 --export` 같은 명령이 성공하며 파일명은 `recent_5000`이지만 실제 결과는 1232회일 수 있다.

결과 본문에는 실제 회차 수와 범위가 있어 계산 자체는 잘못되지 않지만, 자동화나 파일명만 보는 사용자가 범위를 오해할 수 있다.

권장 보완:

* 단일 기간 Service에서 `recent > available`이면 명시적 오류를 반환하거나
* 요청 건수와 실제 적용 건수를 결과에 각각 기록하고 파일명은 실제 범위를 사용한다.

적용 대상은 기본 분석, 간격, 관계, 행렬, 수학 특성, 유사도 명령을 함께 검토한다.

### P3: CLI 파일의 책임이 커졌다

`src/lotto_analysis/cli.py`는 현재 860줄이며 수집, 정제, DB, 모든 분석 명령의 파서·실행 조립·출력 형식을 함께 담당한다. 핵심 계산은 별도 모듈로 잘 분리되어 있지만, 번호 생성과 백테스트 명령까지 같은 파일에 추가하면 충돌과 회귀 위험이 커진다.

권장 보완:

* CLI 진입점과 명령 등록은 유지한다.
* 분석별 실행·출력 코드를 `cli_commands/analysis.py` 같은 모듈로 점진적으로 이동한다.
* 대규모 일괄 재작성 대신 다음 기능을 추가할 때 관련 부분만 이동한다.

### P3: 유사도 계산은 반복 실행 시 비용이 커질 수 있다

유사도는 모든 순서 없는 회차쌍을 비교하므로 시간 복잡도가 O(N²)이다. 현재 1232회, 758296쌍은 정상 실행되지만 백테스트가 매 목표 회차마다 전체 유사도를 다시 계산하면 중복 계산이 급격히 늘어난다.

권장 보완:

* 현재 단발성 분석 구현은 유지한다.
* 번호 생성에서는 과거 조합을 `frozenset`으로 한 번 준비해 재사용한다.
* 백테스트 설계 시 회차가 늘어날 때 증분 비교하거나 캐시하는 방식을 검토한다.

### P3: PostgreSQL 고급 분석 경로의 자동 통합 검증이 없다

순수 계산과 Service 범위 전달은 단위 테스트로 검증되고 실제 PostgreSQL CLI도 수동 실행했다. 그러나 선택적 PostgreSQL 통합 테스트는 Repository upsert/read만 확인하고 `relationships`, `matrix`, `patterns`, `similarity` Service 결과의 합계 불변식은 자동 검사하지 않는다.

권장 보완:

* 기존 `postgres` 마커 통합 테스트에 실제 저장 데이터를 이용한 최소 불변식 검사를 추가한다.
* 전체 결과값을 고정하지 말고 `회차 수×6`, 조합 수, 분포 합계 같은 안정적인 조건만 검사한다.

## 확인한 불변식

전체 PostgreSQL 1232회 기준:

* 번호쌍 합계: `18480 = 1232×15`
* 3개 조합 합계: `24640 = 1232×20`
* 번호 거리 관측 합계: `18480 = 1232×15`

* 인접 간격 합계: `6160 = 1232×5`
