# FastAPI 핵심 범위 코드 리뷰

검토일: 2026-07-18

## 결론

FastAPI 핵심 범위는 기존 Repository·Service 계층을 재사용하며 Streamlit 또는 CLI의 계산을 복사하지 않는다. 명시적 요청·응답 스키마, OpenAPI, 도메인 오류와 DB 오류 처리가 갖춰져 Next.js 연결 전 기반으로 사용할 수 있다.

리뷰에서 확인한 P1·P2 항목은 이번 보완에서 해결했다. 인증·비동기 작업 큐·캐시는 실제 배포 및 사용자 요구가 정해지기 전에는 추가하지 않는다.

## 해결한 항목

### P1: SQLAlchemy 엔진 종료 누락

프로세스 전역으로 캐시한 엔진이 앱 종료 시 해제되지 않았다. FastAPI lifespan에서 생성된 엔진만 `dispose()`하고 캐시를 비우도록 수정했으며 단위 테스트를 추가했다.

### P1: 동기 백테스트 실행량 무제한

반복 실험의 seed·조합 목록과 값에 충분한 상한이 없어 요청 하나가 서버를 장시간 점유할 수 있었다. 단일 백테스트는 최대 5,000개 후보 평가 단위, 반복 실험은 최대 100,000개 생성 조합 단위로 제한했다. 목록 길이·중복과 개별 조합 수도 검증한다.

### P2: Next.js 개발 CORS 미설정

기본적으로 localhost와 127.0.0.1의 3000 포트를 허용하고 `LOTTO_CORS_ORIGINS`로 교체할 수 있게 했다. 허용 메서드는 GET·POST·OPTIONS이며 자격 증명은 아직 허용하지 않는다.

### P2: DB 오류 응답 자동 검증 누락

SQLAlchemy 오류가 접속 상세를 노출하지 않는 고정 503 응답으로 변환되는지 테스트를 추가했다.

## 후속 P3

* 여러 사용자가 동시에 고비용 백테스트를 실행하는 단계에서는 작업 큐와 비동기 상태 조회가 필요하다.
* 전체 `/api/draws` 응답은 현재 1232회 규모에서는 사용할 수 있지만 Next.js 요구에 맞춰 페이지네이션을 검토한다.
* API 계약 테스트가 더 늘어나면 분석·생성·백테스트 파일로 분리한다.
* 실제 PostgreSQL을 사용하는 전체 API 통합 테스트는 선택적 테스트로 확대할 수 있다.

## 판단

현재 로컬 개발과 단일 사용자 내부 서비스 범위에서는 FastAPI 단계를 마감할 수 있다. 다음 제품 단계는 Next.js 요구사항을 먼저 확정한 뒤 API 페이지네이션·CORS 운영 origin·인증 필요성을 결정하는 것이다.
