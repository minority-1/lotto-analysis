# Next.js 사용자 화면 전체 코드 리뷰

검토일: 2026-07-20

검토 범위: 대시보드, 회차, 분석, 번호 생성, 단일·반복 백테스트와 공통 API·탐색·반응형 스타일.

## 요약

FastAPI를 통해서만 백엔드 기능을 사용하는 아키텍처와 과거 통계를 예측으로 표현하지 않는 원칙은 일관되게 유지되어 있다. 핵심 사용자 경로는 모두 연결됐으며 정적 검사, 프로덕션 빌드와 Python 회귀 테스트가 통과한다.

## 보완 결과

### P1 완료: 회차 상세의 404와 API 장애 구분

기존 구현은 단일 회차의 404와 FastAPI·PostgreSQL 장애를 모두 `notFound()`로 처리했다. API 오류에 HTTP 상태를 보존하고, 실제 404만 없는 회차로 처리하며 나머지는 서버 상태 안내 화면을 표시하도록 수정했다.

### P1 완료: 모바일 주요 메뉴 복구

850px 이하에서 공통 주요 메뉴가 완전히 숨겨져 화면 간 이동이 불가능했다. 헤더를 두 줄 구조로 전환하고 주요 메뉴를 가로 스크롤 가능한 탐색으로 유지했다.

### P2 완료: 고정 연결 상태 문구 제거

연결 확인 없이 항상 `API 연결됨`을 표시하던 문구를 상태를 단정하지 않는 `FASTAPI 연동`으로 변경했다. 실제 장애 여부는 각 데이터 화면의 오류 상태에서 안내한다.

## 후속 항목

### P2: Server Action 입력 상태 보존

번호 생성과 백테스트 폼은 실행 후 브라우저가 uncontrolled form을 초기화할 수 있다. 결과 재현과 조건 수정 흐름을 위해 제출 요청값을 action state에 보존하고 폼을 복원하는 공통 방식을 적용할 필요가 있다.

### P3: 프런트엔드 자동 테스트

현재 ESLint, TypeScript, 프로덕션 빌드와 실제 브라우저 확인은 수행하지만 컴포넌트·E2E 테스트는 없다. 화면 구조가 더 안정된 뒤 핵심 경로 중심의 Playwright 테스트 도입을 검토한다.

### P3: 배포 환경 관측성

현재는 로컬 개발이 기준이다. 배포 시 요청 ID, 구조화된 프런트 오류 기록, 상태 점검 경로와 타임아웃 정책을 확정해야 한다.
