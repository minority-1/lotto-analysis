# Python 기반 로또 데이터 수집·통계 분석·번호 조합 생성 프로젝트

당신은 Python 데이터 엔지니어, 통계 분석가, FastAPI 백엔드 개발자이자 Next.js 프런트엔드 설계자입니다.

저와 함께 한국 로또 6/45 당첨 데이터를 수집하고 저장한 뒤, 다양한 통계 분석과 시각화를 제공하고, 사용자가 설정한 조건에 따라 번호 조합을 생성하는 프로젝트를 단계적으로 개발해주세요.

이 프로젝트는 단순 예제나 일회성 스크립트가 아니라, 실제로 실행하고 기능을 확장할 수 있는 구조를 목표로 합니다.

한 번에 전체 프로젝트를 완성하려 하지 말고, 먼저 설계와 요구사항을 함께 검토한 뒤 단계별로 구현해주세요.

---

## 1. 프로젝트의 핵심 목표

프로젝트는 다음 흐름으로 구성합니다.

```text
로또 데이터 수집
→ 원본 파일 저장
→ 데이터 정제 및 유효성 검사
→ 데이터베이스 저장
→ 기본 통계 분석
→ 고급 패턴 분석
→ 분석 결과 시각화
→ 조건 기반 번호 조합 생성
→ 과거 회차 백테스트
→ FastAPI 제공
→ Next.js 사용자 UI
```

초기에는 Python만으로 빠르게 기능을 검증합니다.

```text
Streamlit
→ Python Service
→ SQLite
```

최종적으로는 다음 구조로 발전시킵니다.

```text
Next.js
→ FastAPI
→ Python 분석 및 서비스 모듈
→ PostgreSQL
```

Python 백엔드와 분석 엔진은 최종 단계에서도 유지합니다.

Next.js는 최종 사용자 화면을 담당하고, Streamlit은 초기 분석 화면 또는 내부 관리자·연구용 도구로 사용할 수 있습니다.

## 2. 프로젝트의 성격

이 프로젝트의 우선순위는 다음과 같습니다.

1. 데이터 수집과 데이터 정확성
2. 통계 분석
3. 분석 결과 시각화
4. 조건 기반 번호 조합 생성
5. 백테스트와 전략 비교
6. 최종 웹서비스 구성

번호 생성 기능은 미래 당첨번호를 정확히 예측하거나 당첨을 보장하는 기능이 아닙니다.

과거 데이터를 활용하여 사용자가 지정한 통계적 조건을 만족하는 후보 조합을 생성하고, 생성 전략별 특징과 과거 결과를 비교하는 기능으로 정의합니다.

통계 점수가 높다는 이유만으로 실제 당첨 확률이 높아진다고 표현하지 마세요.

```text
│   ├── schemas/
```

## 5. 데이터 저장 단계 분리

데이터는 다음 단계로 분리합니다.

### 원본 데이터

수집 당시의 응답을 최대한 그대로 보관합니다.

```text
data/raw/
```

기본 형식은 JSON을 고려합니다.

원본 데이터는 다음 상황에서 활용합니다.

* 사이트 응답 구조 변경 확인
* 잘못 정제된 데이터 복구
* 특정 회차 재검증
* 수집 오류 분석

### 정제 데이터

분석과 DB 저장에 적합하도록 정제합니다.

```text
data/processed/lotto_draws.csv
```

기본 형식은 CSV로 시작합니다.

필요하면 이후 Parquet을 추가할 수 있게 설계해주세요.

### 분석 데이터

합계, 홀짝 개수, 구간 분포 등 파생값이 포함된 데이터입니다.

```text
data/analysis/
```

원본 당첨 데이터와 파생 분석 데이터를 무조건 같은 테이블에 저장하지 말고, 계산 비용과 사용 빈도를 검토하여 분리 여부를 제안해주세요.

## 6. 데이터 수집 기능

가능하면 공식 데이터 출처를 우선 사용합니다.

공식 API나 안정적인 JSON 응답이 존재하면 우선 검토하고, 그렇지 않으면 HTML 파싱 방식을 사용합니다.

수집 대상의 기본 항목은 다음과 같습니다.

* 회차 번호
* 추첨일
* 당첨번호 6개
* 보너스 번호
* 1등 당첨자 수
* 1등 당첨금
* 총 판매금액
* 수집일시

권장 컬럼명:

```text
draw_number
draw_date
num1
num2
num3
num4
num5
num6
bonus_number
first_prize_winners
first_prize_amount
total_sales_amount
collected_at
```

수집 방식은 다음 네 가지 모드를 지원하도록 설계합니다.

### 전체 수집

```text
1회차부터 최신 회차까지 수집
```

### 증분 수집

```text
현재 저장된 마지막 회차 이후만 수집
```

### 누락 보완

```text
저장된 전체 범위를 검사하여 누락된 회차만 수집
```

### 특정 범위 재수집

```text
사용자가 지정한 시작 회차부터 종료 회차까지 다시 수집
```

수집 결과 상태는 다음과 같이 구분해주세요.

```text
신규 수집
기존 데이터와 동일
기존 데이터와 다름
수집 실패
응답 데이터 없음
응답 형식 오류
유효성 검사 실패
```

다음 기능도 고려해주세요.

* 요청 간격 적용
* 일시적 오류 재시도
* 최대 재시도 횟수
* 실패 회차 별도 기록
* 진행 상태 출력
* 로그 파일 저장
* 중단 후 재실행
* 최신 회차 확인
* 변경된 데이터 비교
* 선택적 업데이트

## 7. 데이터 유효성 검사

다음 규칙을 기본으로 합니다.

* 회차 번호는 양의 정수
* 일반 당첨번호는 1부터 45 사이
* 일반 당첨번호 6개는 서로 중복될 수 없음
* 보너스 번호는 1부터 45 사이
* 보너스 번호는 일반 당첨번호와 중복될 수 없음
* 일반 당첨번호는 저장 시 오름차순 정렬
* 동일한 회차는 중복될 수 없음
* 추첨일은 필수
* 금액 데이터는 숫자로 변환
* 중간에 누락된 회차 확인
* 잘못된 데이터는 정상 데이터와 분리
* 파일 데이터와 DB 데이터 건수 비교
* 파일과 DB 사이의 회차 누락 비교

검증 결과에는 다음 내용이 포함되어야 합니다.

```text
전체 데이터 건수
정상 데이터 건수
오류 데이터 건수
중복 회차
누락 회차
오류 항목
오류 사유
검증 실행 시간
결과 파일 위치
```

## 9. 통계 분석 범위

분석 기능은 한 번에 전부 구현하지 말고 단계별로 나눕니다.

### 1단계: 기본 분포 분석

* 번호별 전체 출현 횟수
* 번호별 출현 비율
* 최근 5회 빈도
* 최근 10회 빈도
* 최근 20회 빈도
* 최근 30회 빈도
* 최근 50회 빈도
* 최근 100회 빈도
* 번호별 마지막 출현 회차
* 번호별 현재 미출현 기간
* 홀수·짝수 분포
* 낮은 번호·높은 번호 분포
* 번호 합계
* 평균
* 중앙값
* 표준편차
* 구간별 번호 분포
* 끝수 분포
* 연속번호 포함 여부
* 직전 회차 번호 재출현 개수

낮은 번호와 높은 번호의 기준은 임의로 정하지 말고 먼저 제안해주세요.

예:

```text
1~22 / 23~45
```

구간 구분도 다음 두 방식 등을 비교하여 제안해주세요.

```text
1~9 / 10~19 / 20~29 / 30~39 / 40~45
```

```text
1~10 / 11~20 / 21~30 / 31~40 / 41~45
```

### 2단계: 관계 분석

* 번호쌍 동시 출현 횟수
* 번호 3개 조합 동시 출현 횟수
* 특정 번호와 함께 나온 번호
* 번호 간 거리
* 당첨번호 ±1 관계
* 직전 회차와 중복
* 2회 전 데이터와 중복
* 3회 전 데이터와 중복
* 보너스 번호의 이후 일반번호 출현
* 연속번호 그룹
* 같은 끝수 관계

### 3단계: 고급 패턴 분석

* 7×7 번호 행렬
* 행별 출현 개수
* 열별 출현 개수
* 대각선 패턴
* 최근 N회 행렬
* 기간별 행렬 비교
* 두 기간의 출현 차이 행렬
* AC 값
* 번호 간격 분포
* 소수 개수
* 합성수 개수
* 제곱수 포함 여부
* 번호 합계 구간
* 끝수 합계
* 과거 당첨 조합과의 유사도

각 분석 기능은 독립적인 함수 또는 클래스로 작성합니다.

분석 결과는 가능하면 pandas DataFrame 또는 직렬화 가능한 모델로 반환해주세요.

```text
전체 회차
최근 5회
최근 10회
최근 20회
최근 30회
최근 50회
최근 100회
최근 1년
특정 연도
사용자 지정 회차 범위
사용자 지정 날짜 범위
```

예:


## 11. 출현 간격 분석

‘주기 예측’이라는 표현보다 객관적인 ‘출현 간격 분석’을 우선 구현합니다.

번호별로 다음 값을 계산합니다.

* 출현 회차 목록
* 각 출현 사이의 간격
* 평균 출현 간격
* 중앙 출현 간격
* 최소 출현 간격
* 최대 출현 간격
* 최근 출현 간격
* 현재 미출현 기간
* 표준편차

평균 간격이 다음 출현을 보장하는 것처럼 설명하지 마세요.

출현 간격은 과거 데이터의 분포를 설명하는 통계로만 사용합니다.

## 12. 번호 조합 생성 구조

번호 생성 기능에서는 ‘생성 전략’과 ‘조합 조건’을 반드시 분리합니다.

### 생성 전략

번호 후보를 어떤 방식으로 선택할지 결정합니다.

초기 전략 후보:

```text
완전 무작위
전체 빈도 가중치
최근 빈도 가중치
장기 미출현 번호 가중치
전체 빈도와 최근 빈도 혼합
고빈도와 저빈도 혼합
사용자 정의 점수 기반
```

가중치 기반 방식에서는 출현 횟수를 그대로 확률로 사용하지 말고, 특정 번호에 확률이 지나치게 집중되지 않도록 정규화, 스무딩 또는 가중치 제한 방식을 제안해주세요.

### 조합 필터

생성된 조합이 사용자의 조건을 만족하는지 검사합니다.

조건 후보:

* 생성할 조합 개수
* 반드시 포함할 번호
* 제외할 번호
* 홀수 개수 범위
* 짝수 개수 범위
* 낮은 번호 개수
* 높은 번호 개수
* 번호 합계 범위
* 구간별 최소·최대 개수
* 연속번호 최대 개수
* 연속번호 그룹 수
* 동일 끝수 최대 개수
* 소수 개수 범위
* 직전 회차 번호 포함 개수
* 최근 N회 고빈도 번호 포함 개수
* 장기 미출현 번호 포함 개수
* 과거 1등 번호와 완전히 같은 조합 제외
* 과거 당첨 조합과 5개 이상 같은 조합 제외
* 생성 결과끼리 지나치게 유사한 조합 제외

생성 흐름은 다음 구조를 따릅니다.

```text
생성 전략 선택
→ 번호 후보 생성
→ 조합 조건 검사
→ 조합 점수 계산
→ 결과 정렬
→ 저장 또는 내보내기
```

조건이 너무 엄격하여 조합을 만들기 어려운 경우 무한 반복하지 않도록 최대 시도 횟수를 둡니다.

생성하지 못한 경우 어떤 조건이 충돌했을 가능성이 있는지 사용자에게 안내해주세요.

재현 가능한 테스트를 위해 선택적으로 random seed를 받을 수 있게 합니다.

## 13. 조합 설명과 점수

생성 결과는 번호만 반환하지 말고 조합의 통계적 특징을 함께 반환합니다.

예:

```text
추천 조합: 3, 11, 18, 27, 34, 42

생성 전략: 최근 빈도와 장기 미출현 혼합
홀짝: 3:3
고저: 3:3
합계: 135
연속번호: 없음
동일 끝수: 최대 1개
최근 30회 고빈도 번호: 11, 34
장기 미출현 번호: 18
직전 회차 중복: 1개
AC 값: 8
```

점수 후보:

* 전체 빈도 점수
* 최근 빈도 점수
* 미출현 기간 점수
* 번호쌍 관계 점수
* 홀짝 균형 점수
* 고저 균형 점수
* 합계 구간 점수
* 번호 간격 점수
* 연속번호 점수
* 끝수 분산 점수
* 과거 당첨 조합 차별성 점수

점수는 사용자가 설정한 통계적 선호 조건을 얼마나 만족하는지를 표현하는 값입니다.

실제 당첨 가능성을 의미하지 않습니다.

점수의 계산 공식과 가중치를 숨기지 말고 설명해주세요.

## 14. 백테스트

번호 생성 전략을 과거 회차에 적용하여 비교합니다.

가장 중요한 원칙은 미래 데이터 누수 방지입니다.

예를 들어 1000회차를 테스트한다면:

```text
1회부터 999회까지의 데이터만 분석
→ 1000회차 후보 조합 생성
→ 실제 1000회차 결과와 비교
```

1000회차 데이터를 통계 계산에 포함하면 안 됩니다.

백테스트 과정:

1. 테스트 대상 회차 선택
2. 대상 회차 이전 데이터만 조회
3. 통계 정보 계산
4. 지정 전략으로 후보 조합 생성
5. 실제 당첨번호와 비교
6. 결과 저장
7. 여러 회차에 반복
8. 전략별 결과 비교

기록 항목:

```text
기준 회차
예측 대상 회차
학습에 사용한 시작 회차
학습에 사용한 종료 회차
생성 전략
생성 조건
생성 조합
실제 당첨번호
일치 개수
보너스 번호 일치 여부
조합 점수
random seed
실행 시간
```

비교할 전략:

```text
완전 무작위
조건만 적용한 무작위
통계 가중치 기반
혼합 전략
사용자 정의 전략
```

모든 전략은 동일한 회차 수와 동일한 조합 생성 개수로 비교해주세요.

다음 조건도 구분하여 결과를 집계합니다.

```text
회차당 1조합
회차당 5조합
회차당 10조합
회차당 50조합
```

조합 수가 많아져 일치 확률이 올라간 결과를 전략의 성능 향상으로 오해하지 않도록 해석해주세요.

## 16. FastAPI와 Next.js 전환

Python 분석 기능이 안정화되면 FastAPI를 추가합니다.

FastAPI는 기존 Service 계층을 호출하고, 분석 로직을 중복해서 구현하지 않습니다.

예상 API 영역:

```text
/api/draws
/api/collection
/api/analysis/frequency
/api/analysis/intervals
/api/analysis/pairs
/api/analysis/matrix
/api/combinations/generate
/api/backtests
```

응답은 Next.js에서 사용하기 쉬운 JSON 구조로 제공합니다.

DataFrame을 그대로 응답하지 말고 명확한 응답 스키마로 변환해주세요.

최종 Next.js 화면은 FastAPI를 통해 데이터를 조회합니다.

```text
Next.js
→ FastAPI
→ Service
→ Repository
→ PostgreSQL
```

Next.js에서 Python 분석 코드를 다시 구현하지 마세요.

Streamlit은 이후에도 내부 관리 및 분석 실험 도구로 유지할 수 있습니다.

## 20. 대화 및 답변 방식

가장 중요한 요청입니다.

처음부터 전체 프로젝트 코드나 완성된 파일을 한 번에 출력하지 마세요.

먼저 현재 요구사항을 분석한 뒤 다음 내용을 제시해주세요.

1. 현재까지 확정된 내용
2. 추가로 결정해야 할 사항
3. 개선하거나 주의할 부분
4. 추천 아키텍처
5. 1차 개발 범위
6. 1차 단계에서 생성할 파일 목록

확정되지 않은 내용은 임의로 결정하지 말고, 추천안과 이유를 제시해주세요.

다만 사소한 구현 세부사항까지 매번 질문하지 말고, 일반적으로 안전한 기본값은 제안한 뒤 진행할 수 있게 해주세요.

제가 특정 단계의 구현을 요청하면 해당 단계에 필요한 코드만 작성해주세요.

각 단계의 답변에는 다음 내용을 포함해주세요.

* 단계의 목적
* 설계 설명
* 생성 또는 수정할 파일 목록
* 각 파일의 전체 경로
* 실행 가능한 전체 코드
* 설치할 패키지
* 실행 명령어
* 예상 실행 결과
* 테스트 방법
* 주의사항
* 다음 단계와의 연결 관계

코드 중간을 생략하거나 다음과 같이 작성하지 마세요.

```text
나머지는 동일
일부 생략
기존 코드 사용
필요에 따라 구현
```
